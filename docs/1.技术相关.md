# S.L.O.V.E. 系统技术实现白皮书

本文档旨在详细解析 S.L.O.V.E (视频推荐系统 Demo) 的系统架构、核心推荐算法、特征可视化逻辑以及工程交互链路。

## 1. 系统架构与技术栈

本系统采用经典的前后端分离结构，结合机器学习模型，构建了一个无需用户登录体系也可以实现个性化推荐的轻量级闭环系统。

### 技术选型

* **后端服务**: 基于 Python 的 [Flask](https://flask.palletsprojects.com/) 轻量级 Web 框架，提供 RESTful API 接口。
* **算法引擎**: [scikit-learn](https://scikit-learn.org/)，核心应用于自然语言文本的特征提取与矩阵降维分析。
* **前端展示**: 原生 HTML5 / CSS3 / JavaScript，UI 视觉由赛博朋克科幻风格组成；数据可视化部分基于 [Apache ECharts 5](https://echarts.apache.org/)。
* **客户端存储**: 利用浏览器 `localStorage` 进行用户观影痕迹的长效追踪。

## 2. 核心推荐算法 (Content-Based Recommendation)

系统推荐模型采用**基于内容的推荐过滤 (Content-Based Filtering)** 原理，将非结构化的自然语言信息转化为数学向量并在高维空间中进行相似度检索。

### 2.1 文本特征向量化 (TF-IDF)

视频数据集包含了标题、标签分类等文本元数据。系统使用 TF-IDF (词频-逆文档频率) 算法来量化一段文本中某个词对该文本的重要程度：
$$ \text{TFidf}(t, d, D) = \text{tf}(t, d) \cdot \text{idf}(t, D) $$

* **TF (词频)**: 特征词 $t$ 在当前视频文档 $d$ 中出现的频率。
* **IDF (逆文档频率)**: 如果一个特定词 (如 "视频") 在所有被处理文档 $D$ 中都很常见，它的重要度会被惩罚性降低；如果是独有的特性词 (如 "赛博朋克")，则重要度提升。
通过 `TfidfVectorizer`，系统把全部的视频数据映射为了一个 $N$ 维特征矩阵环境。

### 2.2 用户画像构建 (User Embedding)

为保护隐私，系统不依赖云端账户，而是通过前端本地收集用户的观影序列。
当后端接收到用户的观看历史列表后：

1. 过滤检索出该用户看过的所有视频记录。
2. 提取这些视频在 $N$ 维矩阵中对应的向量数组。
3. 采用**特征加权平均 (Mean Pooling)** 的策略，将用户历史浏览对应的多条视频向量融合成一条**中心用户向量**，它代表了用户的当前偏好重心。

### 2.3 余弦相似度计算 (Cosine Similarity)

系统不需要去计算高维空间的欧氏距离，而是通过判断这根**中心用户向量**与待推荐**候选视频向量**之间的空间夹角 (余弦相似度)，决定内容的推荐优先级。
$$ \text{similarity} = \cos(\theta) = \frac{A \cdot B}{||A|| \times ||B||} $$
相似度的取值在 $[0, 1]$ 之间。夹角越小（余弦值越趋近于1），说明视频的标签内容与该用户的偏好越吻合，排序也就越靠前。

## 3. 高维空间数据可视化 (PCA 降维)

在推荐页面，为了直观地展示机器是如何进行学习推荐的，系统引入了**数据可视化仪表盘**。但在机器学习模型中，一篇文章的特征往往达到了几百乃至上千个维度，人类肉眼只能观察到二至三维空间的设计。

在这个场景中，采用 **主成分分析算法 (PCA - Principal Component Analysis)** 来解决该痛点：

* **降维转换**: PCA通过寻找数据方差最大的方向，将原始高维稀疏特征的矩阵（如 TF-IDF 生成的稀疏矩阵）投射到低维的平面二维空间上 (即 $X$ 和 $Y$ 坐标轴)。
* **前端渲染**: 降维计算完成后，数据由 Flask API 传往前端并绑定到 ECharts 实例上。
  * **视频节点 (`scatter`)**: 以散点落位于坐标系内，一旦被系统列为推荐内容，节点状态便会被高亮染色。
  * **用户节点 (`scatter`)**: 代表用户偏好矢量的坐标拥有更高辨识度的光环与大小。

此时可以通过图表肉眼直观地证明这套算法的心象：**用户的多维特征由于属于偏好重心，因此在图中始终位于其历史喜好的视频点群重心附近**。并且那些在坐标投影内距离用户中心极其相近的视频节点，正是被系统高分推送的候选池内容。这就解释了系统之所以能够“猜你喜欢”的底层数学逻辑。

## 4. API 接口交互设计

系统交互链路以轻量化的 RESTful JSON 通信作为桥梁。

| 路由接口 | 请求方式 | 作用说明 |
| - | - | - |
| `/api/videos` | GET | 返回系统数据库中全量视频初始化数据，用于视频大厅基础渲染。 |
| `/api/recommend` | POST | 接收 `{"history": [...]}` 进行实时推荐推演，返回经过余弦相似度降序排序的 Top N 系列推荐视频信息。 |
| `/api/visualize` | POST | 接收 `{"history": [...]}` 历史数组，将 Tfidf 矩阵与 User 矩阵合并运行 `PCA` 算法完成二维世界转换，将生成的 `(x,y)` 坐标地图发送给前端 ECharts 进行散点作数。 |

---

## 5. 项目工程总结

S.L.O.V.E 不涉及复杂的大型深度学习网络构建或分布式算力部署，是一套非常典型的、针对零启动(Cold Start)用户友好的“本地化内容基”轻服务展示系统。它的空间建模在每次用户点开任意视频加载时都会利用后台算力完成动态推导与结果更新。

借助于数学 PCA 降维模型与前端 ECharts WebGL 动画结合，能够将原先藏在黑盒里的枯燥“AI 计算排序过程”，非常清晰、透明且具有极客视觉艺术感地传达给了使用者。
